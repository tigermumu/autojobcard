## 四、提高匹配成功率的建议

### 4.1 关键词字典优化（最重要）

#### 需要添加到关键词字典的主部件/子部件
```
主部件层级：
- WORKTOP
- CLOSET
- STOWAGE
- DIVIDER
- BAR

子部件层级：
- TRIM (VERTICAL TRIM, EDGE TRIM, PLASTIC TRIM)
- RETAINER (TURN RETAINER)
- SEAL (TOP SEAL, LOWER SEAL)
- LATCH
- BUMPER
- HINGE
- RAIL
- DRAWER
- RETRIEVER
- FOOT STEP
- TEE GUIDE
- GRILL
- CURTAIN
- DRAIN HOSE
```

#### 需要添加的状态词
```
- DEFORMATION / DEFORMED (变形)
- CORROSION (腐蚀)
- JAMMED (卡住)
- AGING (老化)
- RUST (生锈)
- STRIPPED (滑牙)
```

### 4.2 同义词/近义词扩展

| 原词 | 同义词/近义词 |
|------|--------------|
| WORKTOP | WORK SURFACE, COUNTER TOP, WORKTABLE |
| WORN | DAMAGED, DETERIORATED |
| BROKEN | CRACKED, FRACTURED |
| DENT | DENTED, INDENTED |
| FAILED | DEFECTIVE, MALFUNCTION |
| TRIM | MOLDING, EDGING |

### 4.3 匹配算法优化建议

#### 4.3.1 降低精确匹配要求
当前匹配要求子部件精确匹配，建议：
- 支持部分匹配（如 "VERTICAL TRIM" 匹配 "TRIM"）
- 支持词根匹配（如 "DENTED" 匹配 "DENT"）

#### 4.3.2 增加模糊匹配能力
```python
# 建议添加的匹配规则
- 忽略位置后缀（如 LH, RH, FWD, AFT）后再匹配
- 忽略数量信息（如 QTY:2EA）后再匹配
- 支持子字符串匹配
```

#### 4.3.3 调整评分权重
当前子部件权重太高（46.5分），如果缺少子部件匹配则很难达到阈值：
- 建议在子部件缺失时，增加主部件和状态词的权重
- 或者降低子部件匹配的权重比例

#### 4.3.4 增加备选匹配机制
当精确匹配失败时，尝试：
1. 只用主部件 + 状态词匹配
2. 只用主部件 + 位置匹配
3. 使用文本相似度匹配

### 4.4 工卡库补充

根据失败记录分析，建议补充以下类型的工卡：
1. **WORKTOP/WORKTABLE 相关工卡**（10条失败记录）
2. **CLOSET 相关工卡**（8条失败记录）
3. **RETAINER 相关工卡**（8条失败记录）
4. **TRIM 相关工卡**（13条失败记录）
5. **特殊设备工卡**（ESPRESSO MAKER, TRASH COMPACTOR 等）

### 4.5 数据预处理优化

#### 4.5.1 标准化处理
```python
# 在匹配前对描述进行标准化
- 去除 LOC:xxx 等位置信息
- 去除 QTY:xxx 等数量信息
- 去除 REMARK:xxx 等备注信息
- 统一大小写
- 去除多余空格
```

#### 4.5.2 关键词提取优化
```python
# 更智能的关键词提取
- 识别复合词（如 FOLDING TABLE, WORK SURFACE）
- 识别带修饰词的关键词（如 VERTICAL TRIM, UPPER DOOR）
- 保留位置标识（如 M2, F1, A3）用于二次匹配
```
