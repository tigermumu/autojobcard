# 如何执行备份命令

## 方法一：使用PowerShell（推荐）

### 步骤1：打开PowerShell
- 按 `Win + X`，选择"Windows PowerShell"或"终端"
- 或在开始菜单搜索"PowerShell"
- 或在文件资源管理器中，在地址栏输入 `powershell` 后回车

### 步骤2：执行创建目录命令
```powershell
New-Item -ItemType Directory -Path "C:\AI\demo3-backup-vpn-version" -Force
```

### 步骤3：执行复制命令
```powershell
robocopy C:\AI\demo3 C:\AI\demo3-backup-vpn-version /E /XD node_modules venv __pycache__ .git exports storage /XF *.pyc *.log *.tmp
```

## 命令说明

### PowerShell命令
```powershell
New-Item -ItemType Directory -Path "C:\AI\demo3-backup-vpn-version" -Force
```
- `New-Item`: 创建新项目（文件或目录）
- `-ItemType Directory`: 指定创建的是目录
- `-Path`: 指定路径
- `-Force`: 如果目录已存在，强制创建（不会报错）

### CMD命令（等价）
```cmd
mkdir C:\AI\demo3-backup-vpn-version
```
- `mkdir`: 创建目录（make directory的缩写）
- 如果目录已存在会提示错误，可以加 `/D` 参数

## 常见问题

### Q: 提示"无法识别命令"
A: 可能是：
- 在CMD中执行了PowerShell命令 → 改用CMD命令或切换到PowerShell
- 命令拼写错误 → 检查命令是否正确

### Q: 提示"拒绝访问"
A: 可能是权限不足：
- 右键PowerShell/CMD，选择"以管理员身份运行"
- 或检查目标目录的权限

### Q: 目录已存在怎么办？
A: 
- PowerShell的 `-Force` 参数会覆盖（不会报错）
- CMD的 `mkdir` 会提示错误，可以删除后重建或使用 `mkdir /D`
