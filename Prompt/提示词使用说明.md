# 工卡数据清洗提示词使用说明

## 文件说明

### 1. 工卡数据清洗提示词_Qwen.md（详细版）
- **格式**：Markdown格式，结构清晰，包含详细说明
- **用途**：完整参考文档，包含所有规则、示例和说明
- **适用场景**：需要详细了解分类规则和注意事项时使用

### 2. 工卡数据清洗提示词_Qwen_简化版.txt（简化版）
- **格式**：纯文本格式，简洁明了
- **用途**：直接用于Qwen API调用的提示词
- **适用场景**：实际数据清洗时使用，可以直接复制到API调用中

## 核心内容

两份提示词都包含以下核心内容：

### 1. 核心分类原则（最重要）
- ⚠️ **绝对禁止**：不应该在结构化的时候把不存在的词和词义也不匹配的词体现在分类中
- ✅ **分类规则**：
  - 词一样：分类结果中的词必须与工卡描述中的词完全一致
  - 词义相似：如果词不完全一样，但词义相似，可以使用
  - 禁止引入不存在的词：绝对不能把工卡描述中不存在的词体现在分类结果中

### 2. 索引表使用规则
- 4级层级结构：主区域 -> 主部件 -> 一级子部件 -> 二级子部件
- 严格匹配：优先在索引表中查找完全匹配的部件名称
- 层级对应：必须按照索引表中的层级关系进行分类

### 3. 特殊处理规则
- 位置信息处理：M718、#1、23D等应放在"位置"字段
- 方向词处理：UPPER、LOWER等根据上下文判断
- 型号信息处理：不应作为部件分类

### 4. 常见错误示例
- 错误：引入不存在的词（如DOOR FRAME，但工卡描述中没有FRAME）
- 正确：使用工卡描述中存在的词（如HINGE）

### 5. 输出格式
- JSON格式
- 包含：主区域、主部件、一级子部件、二级子部件、位置、缺陷主体、缺陷描述

## 使用方法

### 方法1：直接使用简化版提示词

1. 打开 `工卡数据清洗提示词_Qwen_简化版.txt`
2. 复制全部内容
3. 在Qwen API调用中，将提示词作为system message或user message的第一部分
4. 然后添加索引表内容（格式化的索引表结构）
5. 最后添加要清洗的工卡描述

### 方法2：使用详细版作为参考

1. 打开 `工卡数据清洗提示词_Qwen.md` 作为参考
2. 根据实际需要提取关键部分
3. 结合简化版提示词使用

## 注意事项

1. **必须严格遵守核心原则**：词一样或词义相似，禁止引入不存在的词
2. **索引表是参考**：优先使用索引表中的结构，但如果工卡描述中有新部件，可以创建新路径
3. **质量检查**：每次分类后都要检查是否引入了不存在的词
4. **位置信息分离**：位置、型号等信息必须正确分离到"位置"字段
