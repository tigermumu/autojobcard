## 三、需要添加的字段和功能

### 3.1 数据库字段（需要添加）

**表：`import_batch_items`**

| 字段名 | 类型 | 是否可空 | 说明 |
|--------|------|---------|------|
| `candidate_description_en` | `Text` | 是 | 历史工卡描述（英文），来自Excel的 `Candidate Workcard Description (English)` 列 |

### 3.2 前端数据结构（需要添加）

**接口：`MatchResult`**

```typescript
interface MatchResult {
  // ... 现有字段
  candidate_description_en?: string  // 新增：历史工卡描述（英文）
}
```

**接口：`ImportBatchItem`**

```typescript
export interface ImportBatchItem {
  // ... 现有字段
  candidate_description_en?: string | null  // 新增：历史工卡描述（英文）
}
```

### 3.3 需要修改的功能点

1. **Excel导入时**：识别并读取 `Candidate Workcard Description (English)` 列
2. **保存到数据库时**：将 `Candidate Workcard Description (English)` 保存到 `candidate_description_en` 字段
3. **从数据库加载时**：读取 `candidate_description_en` 字段并映射到前端数据结构
4. **导出Excel时**：将 `candidate_description_en` 导出为 `历史工卡描述` 列

## 五、注意事项

1. **字段命名**：数据库字段使用下划线命名（`candidate_description_en`），Excel列名使用中文（`历史工卡描述`）
2. **类型转换**：Excel导入时，数字类型需要转换为字符串（如 `zone_number`、`reference_workcard_item`）
3. **空值处理**：所有可选字段在保存时，空字符串应转换为 `null`
4. **数据来源**：
   - `Candidate Workcard Description (English)` 来自智能匹配模块导出的Excel
   - 该字段表示历史工卡的英文描述，用于记录匹配时的候选工卡描述信息
