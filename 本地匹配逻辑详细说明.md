# 本地匹配逻辑详细说明

## 一、评分权重体系详解

### 1.1 权重分配原理

本地匹配采用**100分制**评分体系，所有维度权重总和为100分。为了引入智能关键词匹配奖励，原权重按**0.93系数**调整，腾出7分给智能匹配。

### 1.2 权重明细表

| 维度 | 原始权重 | 调整后权重 | 计算公式 | 说明 |
|------|---------|-----------|----------|------|
| **main_component** | 35分 | **32.55分** | 35 × 0.93 | 主部件匹配（最重要基础字段） |
| **sub_component** | 50分 | **46.5分** | 50 × 0.93 | 子部件匹配（权重最高） |
| **location** | 3分 | **2.79分** | 3 × 0.93 | 位置匹配 |
| **orientation** | 2分 | **1.86分** | 2 × 0.93 | 方位匹配 |
| **status** | 5分 | **4.65分** | 5 × 0.93 | 状态匹配 |
| **action** | 5分 | **4.65分** | 5 × 0.93 | 动作匹配 |
| **keyword_match_bonus** | 0分 | **7.0分** | 新增 | 智能匹配奖励（新增） |
| **总计** | 100分 | **100分** | - | 权重总和保持100分 |

### 1.3 权重设计理念

1. **子部件权重最高（46.5分）**
   - 子部件是最具体的部件描述，匹配准确性最高
   - 权重接近总分的一半，体现其重要性

2. **主部件权重次之（32.55分）**
   - 主部件是基础匹配字段，必须先匹配
   - 权重约占总分的三分之一

3. **其他维度权重较小**
   - location、orientation、status、action 作为辅助匹配字段
   - 权重较小，但精确匹配时也能贡献分数

4. **智能匹配奖励（7分）**
   - 基于文本相似度的奖励机制
   - 提升匹配准确性，减少误匹配

## 二、主部件匹配策略详解

### 2.1 多值匹配机制

**核心特点**：
- 主部件支持**多值存储**，使用逗号分隔（如："SEAT, BIN, TABLE"）
- 匹配时计算**集合交集**，支持部分匹配

**存储格式**：
```python
# 缺陷主部件
defect.main_component = "SEAT, BIN"

# 工卡主部件
wc.main_component = "SEAT, BIN, TABLE"
```

### 2.2 三种匹配情况详解

#### 情况1：双方都没有主部件

**判断条件**：
```python
if not defect_has_main and not wc_has_main:
    return None  # 不匹配
```

**处理逻辑**：
- 直接返回 `None`，不进行后续匹配
- 这种情况比较罕见，通常表示数据清洗失败

**示例**：
```
缺陷主部件：""（空）
工卡主部件：""（空）
结果：不匹配，返回 None
```

#### 情况2：只有一方有主部件

**判断条件**：
```python
if not defect_has_main or not wc_has_main:
    score_main = 20.0  # 部分匹配
```

**处理逻辑**：
- 给**部分匹配分数 20.0分**
- 虽然只有一方有主部件，但可能是数据不完整，给予部分分数

**示例**：
```
缺陷主部件："SEAT"
工卡主部件：""（空）
结果：20.0分（部分匹配）

缺陷主部件：""（空）
工卡主部件："SEAT, BIN"
结果：20.0分（部分匹配）
```

#### 情况3：双方都有主部件（核心逻辑）

**处理流程**：

**步骤1：转换为集合**
```python
# 将逗号分隔的字符串转换为小写集合
defect_mains = {m.strip().lower() for m in defect.main_component.split(",") if m.strip()}
wc_mains = {m.strip().lower() for m in wc.main_component.split(",") if m.strip()}
```

**步骤2：计算交集**
```python
intersection = defect_mains & wc_mains
match_count = len(intersection)
```

**步骤3：根据匹配数量评分**

| 匹配情况 | 判断条件 | 得分 | 说明 |
|---------|---------|------|------|
| **完全匹配** | `defect_mains == wc_mains` | **32.55分** | 所有关键词都匹配（满分） |
| **多关键词匹配** | `match_count >= 2` | **30.0分** | 匹配2个及以上关键词（高分） |
| **单关键词匹配** | `match_count == 1` | **25.0分** | 匹配1个关键词（中等分） |
| **无交集** | `match_count == 0` | **返回 None** | 不匹配 |

### 2.3 主部件匹配示例

#### 示例1：完全匹配
```
缺陷主部件："SEAT, BIN"
工卡主部件："SEAT, BIN"

转换集合：
- 缺陷：{"seat", "bin"}
- 工卡：{"seat", "bin"}
- 交集：{"seat", "bin"}
- 匹配数：2

判断：defect_mains == wc_mains → True
得分：32.55分（满分）
```

#### 示例2：多关键词匹配
```
缺陷主部件："SEAT, BIN"
工卡主部件："SEAT, BIN, TABLE"

转换集合：
- 缺陷：{"seat", "bin"}
- 工卡：{"seat", "bin", "table"}
- 交集：{"seat", "bin"}
- 匹配数：2

判断：match_count >= 2 → True
得分：30.0分（高分）
```

#### 示例3：单关键词匹配
```
缺陷主部件："SEAT"
工卡主部件："SEAT, BIN, TABLE"

转换集合：
- 缺陷：{"seat"}
- 工卡：{"seat", "bin", "table"}
- 交集：{"seat"}
- 匹配数：1

判断：match_count == 1 → True
得分：25.0分（中等分）
```

#### 示例4：无交集（不匹配）
```
缺陷主部件："SEAT, BIN"
工卡主部件："TABLE, CHAIR"

转换集合：
- 缺陷：{"seat", "bin"}
- 工卡：{"table", "chair"}
- 交集：{}（空集）
- 匹配数：0

判断：match_count == 0 → True
结果：返回 None（不匹配）
```

## 三、子部件匹配策略详解（混合策略）

### 3.1 混合策略设计理念

子部件匹配采用**混合策略**，根据双方是否有子部件，采用不同的处理方式：

1. **双方都没有**：将子部件分数分配给其他匹配字段（避免浪费分数）
2. **双方都有**：按交集匹配（最严格）
3. **只有一方有**：给部分分数（宽容处理）

### 3.2 情况1：双方都没有子部件（分数分配机制）

**判断条件**：
```python
if not defect_has_sub and not wc_has_sub:
    # 分数分配逻辑
```

**核心机制**：将子部件的46.5分分配给其他有匹配的字段

**分配步骤**：

**步骤1：统计有匹配的字段**
```python
matched_fields = []
matched_fields.append("main")  # main总是匹配的，也参与分配
if score_location > 0:
    matched_fields.append("location")
if score_orientation > 0:
    matched_fields.append("orientation")
if score_status > 0:
    matched_fields.append("status")
if score_action > 0:
    matched_fields.append("action")
if score_keyword_match > 0:
    matched_fields.append("keyword_match")
```

**步骤2：计算每个字段的额外分数**
```python
sub_component_weight = 46.5  # 子部件分数上限
bonus_per_field = round(sub_component_weight / len(matched_fields), 2)
```

**步骤3：分配给匹配的字段**
```python
if "main" in matched_fields:
    score_main = round(score_main + bonus_per_field, 2)
if "location" in matched_fields:
    score_location = round(score_location + bonus_per_field, 2)
# ... 其他字段类似
```

**示例**：

假设匹配情况：
- main_component：匹配（得分30.0分）
- location：匹配（得分2.79分）
- orientation：匹配（得分1.86分）
- status：不匹配（得分0分）
- action：不匹配（得分0分）
- keyword_match：匹配（得分3.5分）

**分配计算**：
```
匹配字段：["main", "location", "orientation", "keyword_match"]（4个字段）
子部件分数：46.5分
每个字段额外分数：46.5 ÷ 4 = 11.625分（保留2位小数：11.63分）

最终得分：
- main：30.0 + 11.63 = 41.63分
- location：2.79 + 11.63 = 14.42分
- orientation：1.86 + 11.63 = 13.49分
- keyword_match：3.5 + 11.63 = 15.13分
- sub：0分（已分配给其他字段）
```

**设计目的**：
- 避免因缺少子部件而丢失匹配机会
- 将分数分配给其他有匹配的字段，提高总分

### 3.3 情况2：双方都有子部件（交集匹配）

**判断条件**：
```python
elif defect_has_sub and wc_has_sub:

**处理流程**：

**步骤2：根据匹配数量评分**

| **单关键词匹配** | `match_count == 1` | **41.85分** | 匹配1个关键词（45 × 0.93） |
| **无交集** | `match_count == 0` | **返回 None** | 不匹配 |

**示例1：多关键词匹配**
```

转换集合：

转换集合：

## 四、智能关键词匹配奖励算法详解

### 4.1 算法设计理念

**算法名称**：智能关键词匹配奖励算法 2.0

**设计目标**：
- 基于文本相似度，提升匹配准确性
- 减少因关键词字典不完整导致的误匹配
- 综合考虑覆盖率（Recall）和紧凑度（Precision）

**总分**：7分（占100分总分的7%）

### 4.2 算法流程详解

#### 步骤1：文本分词

**实现代码**：
```python
def get_tokens(text):
    # 使用正则表达式提取所有单词
    # \b\w+\b 匹配单词边界，保留单字母单词（如 P、L、R）
    return set(re.findall(r'\b\w+\b', text.lower()))
```

**分词规则**：
- 使用正则表达式 `\b\w+\b` 提取所有单词
- **保留单字母单词**（如 P、L、R），这些在航空术语中很重要
- 转换为小写并去重（使用集合）

**示例**：
```python
文本："REPLACE SEAT BELT BUCKLE"
分词结果：{"replace", "seat", "belt", "buckle"}

文本："SEAT BELT BROKEN"
分词结果：{"seat", "belt", "broken"}
```

#### 步骤2：计算覆盖率（Recall）- 5分池

**覆盖率定义**：
```
覆盖率 = 交集单词数 / 目标文本单词数
```

**评分规则**：
| 覆盖率 | 得分 | 说明 |
|--------|------|------|
| **≥80%** | **5.0分** | 目标文本的大部分关键词都被匹配 |
| **≥70%** | **3.0分** | 目标文本的大部分关键词被匹配 |
| **<70%** | **0分** | 覆盖率不足 |

**实现代码**：
```python
recall_ratio = len(intersection) / len(target_tokens)
coverage_score = 0.0
if recall_ratio >= 0.8:
    coverage_score = 5.0
elif recall_ratio >= 0.7:
    coverage_score = 3.0
```

**示例1：高覆盖率**
```
目标文本："SEAT BELT BROKEN" → {"seat", "belt", "broken"}（3个词）
候选文本："REPLACE SEAT BELT BUCKLE" → {"replace", "seat", "belt", "buckle"}（4个词）
交集：{"seat", "belt"}（2个词）

覆盖率 = 2/3 = 66.7% → 0分（<70%）
```

**示例2：超高覆盖率**
```
目标文本："SEAT BELT" → {"seat", "belt"}（2个词）
候选文本："REPLACE SEAT BELT BUCKLE" → {"replace", "seat", "belt", "buckle"}（4个词）
交集：{"seat", "belt"}（2个词）

覆盖率 = 2/2 = 100% → 5.0分（≥80%）
```

#### 步骤3：计算紧凑度（Precision）- 2分池

**紧凑度定义**：
```
紧凑度 = 交集单词数 / 候选文本单词数
```

**评分规则**：
- 紧凑度越高，说明候选文本中匹配的单词占比越高
- 得分 = 紧凑度 × 2.0（最高2.0分）

**实现代码**：
```python
precision_ratio = len(intersection) / len(candidate_tokens) if candidate_tokens else 0
compactness_score = precision_ratio * 2.0
```

**示例1：高紧凑度**
```
目标文本："SEAT BELT" → {"seat", "belt"}（2个词）
候选文本："SEAT BELT BROKEN" → {"seat", "belt", "broken"}（3个词）
交集：{"seat", "belt"}（2个词）

紧凑度 = 2/3 = 66.7%
得分 = 66.7% × 2.0 = 1.33分
```

**示例2：低紧凑度**
```
目标文本："SEAT BELT" → {"seat", "belt"}（2个词）
候选文本："REPLACE SEAT BELT BUCKLE STRAP CLIP" → 6个词
交集：{"seat", "belt"}（2个词）

紧凑度 = 2/6 = 33.3%
得分 = 33.3% × 2.0 = 0.67分
```

#### 步骤4：归一化

**归一化公式**：
```python
total_bonus_points = coverage_score + compactness_score
normalized_value = total_bonus_points / 7.0  # 返回0-1范围
final_score = normalized_value * 7.0  # 最终得分（0-7分）
```

**归一化目的**：
- 将总分归一化到0-1范围
- 外部调用时乘以7.0得到最终得分（0-7分）

### 4.3 完整计算示例

#### 示例1：完整匹配

```
目标文本："SEAT BELT BROKEN"
候选文本："REPLACE SEAT BELT BUCKLE"

【步骤1：分词】
- 目标：{"seat", "belt", "broken"}（3个词）
- 候选：{"replace", "seat", "belt", "buckle"}（4个词）
- 交集：{"seat", "belt"}（2个词）

【步骤2：计算覆盖率（Recall）】
覆盖率 = 2/3 = 66.7% → 0分（<70%）

【步骤3：计算紧凑度（Precision）】
紧凑度 = 2/4 = 50%
紧凑度得分 = 50% × 2.0 = 1.0分

【步骤4：归一化】
总奖励分 = 0 + 1.0 = 1.0分
归一化值 = 1.0 / 7.0 = 0.143
最终得分 = 0.143 × 7.0 = 1.0分
```

#### 示例2：高覆盖率匹配

```
目标文本："SEAT BELT"
候选文本："REPLACE SEAT BELT BUCKLE"

【步骤1：分词】
- 目标：{"seat", "belt"}（2个词）
- 候选：{"replace", "seat", "belt", "buckle"}（4个词）
- 交集：{"seat", "belt"}（2个词）

【步骤2：计算覆盖率（Recall）】
覆盖率 = 2/2 = 100% → 5.0分（≥80%）

【步骤3：计算紧凑度（Precision）】
紧凑度 = 2/4 = 50%
紧凑度得分 = 50% × 2.0 = 1.0分

【步骤4：归一化】
总奖励分 = 5.0 + 1.0 = 6.0分
归一化值 = 6.0 / 7.0 = 0.857
最终得分 = 0.857 × 7.0 = 6.0分
```

#### 示例3：完美匹配

```
目标文本："SEAT BELT"
候选文本："SEAT BELT"

【步骤1：分词】
- 目标：{"seat", "belt"}（2个词）
- 候选：{"seat", "belt"}（2个词）
- 交集：{"seat", "belt"}（2个词）

【步骤2：计算覆盖率（Recall）】
覆盖率 = 2/2 = 100% → 5.0分（≥80%）

【步骤3：计算紧凑度（Precision）】
紧凑度 = 2/2 = 100%
紧凑度得分 = 100% × 2.0 = 2.0分

【步骤4：归一化】
总奖励分 = 5.0 + 2.0 = 7.0分
归一化值 = 7.0 / 7.0 = 1.0
最终得分 = 1.0 × 7.0 = 7.0分（满分）
```

### 4.4 算法优势

1. **综合考虑覆盖率（Recall）和紧凑度（Precision）**
   - 覆盖率确保目标文本的关键词被匹配
   - 紧凑度确保候选文本的相关性

2. **保留单字母单词**
   - 在航空术语中，单字母单词（如 P、L、R）很重要
   - 算法保留这些单词，提高匹配准确性

3. **归一化设计**
   - 返回0-1范围，便于外部调用
   - 最终得分乘以7.0得到0-7分

## 七、性能优化机制

### 7.1 主部件索引优化

**优化原理**：
- 在匹配前，按主部件建立索引字典
- 只对相同主部件的缺陷-工卡进行匹配
- 大幅减少匹配次数

**实现方式**：
```python
# 按主部件建立索引
workcard_index = {}
for wc in workcard_cleaned:
    main_key = wc.main_component or ""
    if main_key not in workcard_index:
        workcard_index[main_key] = []
    workcard_index[main_key].append(wc)

# 只匹配相同主部件的工卡
for defect in defect_cleaned:
    defect_main = defect.main_component or ""
    candidate_workcards = workcard_index.get(defect_main, [])
    # 只对 candidate_workcards 进行匹配
```

**性能提升**：
- 假设有1000条缺陷和10000条工卡
- 无索引：需要匹配 1000 × 10000 = 10,000,000 次
- 有索引：假设平均每个主部件有10条工卡，只需匹配 1000 × 10 = 10,000 次
- **性能提升：1000倍**

### 7.2 批量处理优化

**优化原理**：
- 一次性处理所有缺陷记录
- 批量保存匹配结果
- 减少数据库交互次数

**实现方式**：
```python
# 批量保存匹配结果
match_results = []
for defect in defect_cleaned:
    for wc in candidate_workcards:
        score_result = self._score_pair(defect, wc)
        if score_result and score_result["score_total"] >= 85:
            match_results.append({
                "defect_record_id": defect.defect_record_id,
                "workcard_id": wc.workcard_id,
                **score_result
            })

# 批量插入数据库
self.db.bulk_insert_mappings(DefectMatchLocal, match_results)
```

**性能提升**：
- 单条插入：1000次数据库交互
- 批量插入：1次数据库交互
- **性能提升：1000倍**
