# 数据库迁移说明

## 问题诊断
如果保存清洗后的数据失败，很可能是数据库表结构还没有更新。

## 解决方法

### 1. 运行数据库迁移

在 `backend` 目录下运行以下命令：

```bash
# 进入backend目录
cd backend

# 运行迁移
alembic upgrade head
```

或者使用Python：
```bash
python -m alembic upgrade head
```

### 2. 检查当前迁移版本

```bash
alembic current
```

### 3. 如果迁移失败

查看迁移文件：`backend/migrations/versions/003_add_workcard_aircraft_fields.py`

这个迁移会添加以下字段到 `workcards` 表：
- 单机构型识别字段：`aircraft_number`, `aircraft_type`, `msn`, `amm_ipc_eff`
- 清洗后的索引字段：`main_area`, `main_component`, `first_level_subcomponent`, `second_level_subcomponent`, `orientation`, `defect_subject`, `defect_description`, `location_index`, `quantity`
- 原始数据备份字段：`raw_data`

## 错误排查

1. **查看后端日志**：检查详细的错误信息
2. **查看前端控制台**：检查网络请求的错误响应
3. **检查数据库连接**：确保数据库服务正在运行
4. **检查表结构**：确认所有新字段都已创建
