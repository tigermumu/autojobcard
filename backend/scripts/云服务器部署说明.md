# 云服务器构型数据导入指南

## 前置准备

### 1. 确保环境已配置

在云服务器上需要确保：

```bash
# 1. Python环境（Python 3.9+）
python3 --version

# 2. 项目依赖已安装
cd backend
pip3 install -r requirements.txt

# 3. 数据库已创建并运行迁移
alembic upgrade head

# 4. 环境变量配置正确
# 检查 .env 文件中的 DATABASE_URL 配置
```

### 2. 传输导出文件到云服务器

将本地导出的目录传输到云服务器：

```bash
# 方式1：使用 scp（推荐）
scp -r backend/exports/configurations_YYYYMMDD_HHMMSS user@your-server:/path/to/backend/exports/

# 方式2：使用 rsync（更高效）
rsync -avz backend/exports/configurations_YYYYMMDD_HHMMSS user@your-server:/path/to/backend/exports/

# 方式3：打包后传输
# 在本地：
tar -czf configurations_backup.tar.gz backend/exports/configurations_YYYYMMDD_HHMMSS
scp configurations_backup.tar.gz user@your-server:/tmp/

# 在云服务器：
cd /path/to/backend
mkdir -p exports
tar -xzf /tmp/configurations_backup.tar.gz -C exports/
```

## 导入步骤

### 方法1：使用Shell脚本（推荐）

```bash
# 1. 给脚本添加执行权限
chmod +x backend/scripts/import_configurations.sh

# 2. 运行导入脚本
backend/scripts/import_configurations.sh --export-dir backend/exports/configurations_YYYYMMDD_HHMMSS
```

### 方法2：直接使用Python脚本

```bash
cd backend

# 使用最新导出目录（自动查找）
python3 scripts/import_configurations.py

# 或指定导出目录
python3 scripts/import_configurations.py --export-dir exports/configurations_YYYYMMDD_HHMMSS

# 强制使用Python方式（不依赖psql）
python3 scripts/import_configurations.py --use-python --export-dir exports/configurations_YYYYMMDD_HHMMSS
```

### 方法3：使用Python方式（无需PostgreSQL客户端）

如果云服务器没有安装PostgreSQL客户端工具，可以强制使用Python方式：

```bash
cd backend
python3 scripts/import_configurations.py --use-python --export-dir exports/configurations_YYYYMMDD_HHMMSS
```

## 验证导入结果

导入完成后，运行验证脚本：

```bash
cd backend
python3 scripts/verify_configurations.py
```

## 常见问题

### 1. 找不到导出目录

**错误：** `未找到导出目录`

**解决：**
- 检查导出目录路径是否正确
- 使用绝对路径：`--export-dir /full/path/to/exports/configurations_YYYYMMDD_HHMMSS`
- 确保目录存在且有读取权限

### 2. 数据库连接失败

**错误：** `连接数据库失败`

**解决：**
- 检查 `.env` 文件中的 `DATABASE_URL` 配置
- 确保数据库服务正在运行：`sudo systemctl status postgresql`
- 检查防火墙设置
- 验证数据库用户权限

### 3. 权限问题

**错误：** `Permission denied`

**解决：**
```bash
# 给脚本添加执行权限
chmod +x backend/scripts/import_configurations.sh

# 检查文件权限
ls -la backend/exports/configurations_YYYYMMDD_HHMMSS

# 确保uploads目录可写
mkdir -p backend/uploads/index_files
chmod -R 755 backend/uploads
```

### 4. Python模块未找到

**错误：** `ModuleNotFoundError`

**解决：**
```bash
# 确保在正确的虚拟环境中
source venv/bin/activate  # 如果有虚拟环境

# 重新安装依赖
pip3 install -r backend/requirements.txt
```

### 5. 序列重置失败

**警告：** `序列不存在`

**说明：** 这通常不是严重问题，序列会在下次插入数据时自动创建。如果确实需要重置，可以手动执行：

```sql
-- 连接到数据库
psql -U postgres -d aircraft_workcard

-- 重置序列
SELECT setval('configurations_id_seq', COALESCE((SELECT MAX(id) FROM configurations), 1), true);
SELECT setval('index_files_id_seq', COALESCE((SELECT MAX(id) FROM index_files), 1), true);
SELECT setval('index_data_id_seq', COALESCE((SELECT MAX(id) FROM index_data), 1), true);
```

## 完整示例流程

```bash
# 1. SSH登录到云服务器
ssh user@your-server

# 2. 进入项目目录
cd /path/to/your/project

# 3. 激活虚拟环境（如果有）
source venv/bin/activate

# 4. 确保数据库已迁移
cd backend
alembic upgrade head

# 5. 导入数据（使用Python方式，不依赖psql）
python3 scripts/import_configurations.py --use-python --export-dir exports/configurations_20241124_191824

# 6. 验证导入结果
python3 scripts/verify_configurations.py

# 7. 检查数据
# 可以通过Web界面或API检查数据是否正确导入
```

## 注意事项

1. **备份重要数据**：导入前建议备份现有数据库
2. **数据库版本**：确保云服务器的PostgreSQL版本兼容
3. **文件路径**：确保 `uploads/index_files/` 目录存在且有写权限
4. **ID冲突**：如果新服务器已有数据，导入时可能会产生ID冲突，脚本会自动处理
5. **网络连接**：确保云服务器能访问数据库（如果数据库在远程）

## 自动化部署脚本示例

可以创建一个自动化部署脚本：

```bash
#!/bin/bash
# deploy_configurations.sh

EXPORT_DIR=$1

if [ -z "$EXPORT_DIR" ]; then
    echo "用法: $0 <导出目录路径>"
    exit 1
fi

cd backend

echo "开始导入构型数据..."
python3 scripts/import_configurations.py --use-python --export-dir "$EXPORT_DIR"

if [ $? -eq 0 ]; then
    echo "导入成功，开始验证..."
    python3 scripts/verify_configurations.py
else
    echo "导入失败！"
    exit 1
fi
```

使用方法：
```bash
chmod +x deploy_configurations.sh
./deploy_configurations.sh exports/configurations_20241124_191824
```
