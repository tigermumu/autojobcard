## 重要提示

### 国内镜像源配置

为了在腾讯云服务器上获得更好的下载速度和连接可靠性，本指南已配置以下国内镜像源：

- **Ubuntu apt 源**：腾讯云镜像（`mirrors.cloud.tencent.com`）
- **Docker 镜像加速**：腾讯云、中科大、Docker 中国镜像
- **pip 镜像源**：清华大学镜像（`pypi.tuna.tsinghua.edu.cn`）
- **npm 镜像源**：淘宝镜像（`registry.npmmirror.com`）

所有配置已集成在部署步骤中，按照步骤操作即可自动使用国内镜像源。

### Docker Compose 命令说明

**重要**：Ubuntu 22.04+ 使用 `apt install docker-compose-plugin` 安装后，命令是 `docker compose`（没有横线）。

**解决方案**：安装完成后，请执行以下命令创建兼容性脚本，让两种命令都可以使用：

```bash
cat > /usr/local/bin/docker-compose << 'EOF'
#!/bin/bash
exec docker compose "$@"
EOF
chmod +x /usr/local/bin/docker-compose
```

这样 `docker-compose` 和 `docker compose` 都可以正常使用了。

### 项目目录路径说明

**重要**：根据克隆代码的方式，项目可能位于以下两个位置之一：

1. **`/opt/workshop`** - 如果使用 `git clone ... .` 克隆到当前目录
2. **`/opt/workshop/workshop`** - 如果使用 `git clone ...` 克隆到子目录

**如何确认你的项目路径**：
```bash
# 检查项目目录
ls -la /opt/workshop/
# 如果看到 backend/、frontend/、docker-compose.yml 等文件，说明在 /opt/workshop
# 如果看到 workshop/ 目录，说明在 /opt/workshop/workshop
```

**后续所有命令**：请根据你的实际项目路径，将文档中的 `/opt/workshop` 替换为 `/opt/workshop/workshop`（如果适用）。

## 第一步：服务器准备

### 1.1 购买腾讯云 CVM 服务器

1. 登录腾讯云控制台：https://console.cloud.tencent.com/cvm
2. 创建 CVM 实例：
   - **地域**：选择离用户最近的地域
   - **实例规格**：建议至少 2核4GB（根据实际负载调整）
   - **镜像**：选择 **Ubuntu Server 20.04 LTS** 或 **Ubuntu Server 22.04 LTS**
   - **网络**：选择私有网络 VPC
   - **公网 IP**：分配公网 IP
   - **安全组**：创建新的安全组（后续需要配置端口）

### 1.2 配置安全组规则

在腾讯云控制台 → 云服务器 → 安全组，添加入站规则：

| 端口范围 | 授权对象 | 协议类型 | 描述 |
|---------|---------|---------|------|
| 22 | 0.0.0.0/0 | TCP | SSH 远程连接 |
| 80 | 0.0.0.0/0 | TCP | HTTP 访问 |
| 443 | 0.0.0.0/0 | TCP | HTTPS 访问 |

**安全建议**：生产环境建议将 8000 和 3000 端口仅开放给内网或特定 IP。

### 1.3 连接服务器

使用 SSH 客户端连接服务器：

```bash
ssh root@你的服务器公网IP
```

## 第六步：修改 Docker Compose 配置

### 6.1 修改 docker-compose.yml

```bash
cd /opt/workshop  # 或 cd /opt/workshop/workshop
nano docker-compose.yml
```

**重要修改点**：

1. **修改数据库密码**（第7行）：
```yaml
POSTGRES_PASSWORD: 你的强密码
```

2. **修改数据库连接**（第32行）：
```yaml
DATABASE_URL=postgresql://postgres:你的强密码@postgres:5432/aircraft_workcard
```

3. **确认 backend 启动命令**（第30行，应该已存在）：
```yaml
command: python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**完整的 docker-compose.yml 配置示例**：

```yaml
services:
  postgres:
    image: postgres:13
    environment:
      POSTGRES_DB: aircraft_workcard
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: 你的强密码  # 修改这里
    ports:
      - "127.0.0.1:5432:5432"  # 仅本地访问
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - aircraft_network
    restart: unless-stopped

  redis:
    image: redis:6-alpine
    ports:
      - "127.0.0.1:6379:6379"  # 仅本地访问
    volumes:
      - redis_data:/data
    networks:
      - aircraft_network
    restart: unless-stopped

  backend:
    build: ./backend
    ports:
      - "127.0.0.1:8000:8000"  # 仅本地访问，通过 Nginx 代理
    command: python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
    environment:
      - DATABASE_URL=postgresql://postgres:你的强密码@postgres:5432/aircraft_workcard
      - REDIS_URL=redis://redis:6379
      - QWEN_API_KEY=${QWEN_API_KEY}
      - QWEN_BASE_URL=${QWEN_BASE_URL}
      - QWEN_MODEL=${QWEN_MODEL}
    depends_on:
      - postgres
      - redis
    volumes:
      - ./backend:/app
    networks:
      - aircraft_network
    restart: unless-stopped

  frontend:
    build: ./frontend
    ports:
      - "127.0.0.1:3000:3000"  # 仅本地访问，通过 Nginx 代理
    volumes:
      - ./frontend:/app
      - /app/node_modules
    networks:
      - aircraft_network
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:

networks:
  aircraft_network:
    driver: bridge
```

**注意**：Celery 服务已注释，因为项目中尚未实现 celery 应用。如果将来需要，请先创建 `app/celery.py` 文件后再启用。

## 第八步：配置 Nginx 反向代理

### 8.1 创建 Nginx 配置文件

```bash
nano /etc/nginx/sites-available/workshop
```

**推荐配置**（使用公网 IP 访问）：

```nginx
# 前端代理（使用公网 IP 访问）- 推荐配置
server {
    listen 80 default_server;  # default_server 确保这是默认服务器
    server_name _;  # 匹配所有请求

    # 前端页面
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 后端 API（通过 /api 访问）
    # 重要：proxy_pass 后面不要加 /api，让后端自己处理路径
    location /api {
        proxy_pass http://127.0.0.1:8000;  # 注意：这里不要加 /api
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 8.2 启用配置

```bash
# 1. 删除或禁用 Nginx 默认配置（重要！）
rm /etc/nginx/sites-enabled/default

# 2. 启用我们的配置
ln -s /etc/nginx/sites-available/workshop /etc/nginx/sites-enabled/

# 3. 检查是否有其他冲突的配置
ls -la /etc/nginx/sites-enabled/
# 应该只有 workshop，没有 default

# 4. 测试配置
nginx -t

# 5. 如果测试通过，重新加载
systemctl reload nginx

# 6. 验证 Nginx 状态
systemctl status nginx
```

